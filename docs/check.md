# 一、最高优先级：技术内容绝对准确（技术书生命线）

1. 杜绝 “直译导致的技术错误”

   重点核对：技术原理、操作步骤、参数说明、公式释义、代码逻辑、故障排查描述，确保译文和原文技术语义 100% 一致，不增删、不曲解。

   

2. 数据、单位、公式无差错

   

   数字、百分比、计量单位、公式符号严格对照原文，不写错、不换算错（技术书一般保留原单位，无需强行转国标）。

# 二、核心规范：全书术语必须统一（最容易翻车的点）

1. 建立专属术语表

   把高频技术词汇、专业名词、缩写整理成表（前期已经整理过一版，本次核对，先对前期的术语表进行查漏补缺，然后再以此为基础核对其他内容），前后译法完全一致，禁止同一词多种译法。

   例：

   ```
   latency
   ```

    统一 “延迟”，不用 “时延 / 滞后” 混用。

2. 遵循国内技术圈通用译法

   优先采用计算机 / 行业标准译名，不自创生僻译法，避免读者看不懂；

   外来通用名词（如 API、SDK、Docker、Kafka）

   直接保留原文，无需翻译。

3. 缩写首次出现必标注全称

   例：

   ```
   WAF（Web Application Firewall，Web应用防火墙）
   ```

   ，全书标注位置、格式统一。

# 三、语句层面：破除欧化长句，符合中文技术表达

1. 拆分英文长难句

   英文多复合长句，中文直接照搬会晦涩拗口，拆成短句、短句，保证

   逻辑清晰、读起来不费力。

2. 语态适配中文

   英文大量被动语态，译文改为主动语态（技术文档更简洁），例：

   原文被动 → 译文主动：

   ```
   The file is loaded
   ```

    → “加载该文件”，而非 “文件被加载”。

3. 拒绝口语化 / 过度文学化

   技术书追求严谨简洁，不用修饰词、冗余表达，不使用网络用语、抒情句式。

# 四、格式与排版：统一规范，符合出版要求

1. 标点符号严格区分中英文

   中文正文用

   全角标点

   ，代码、英文词汇、数字用

   半角符号

   ，杜绝混排。

2. 标题层级、列表格式统一

   一级 / 二级 / 三级标题样式、序号（1.0 / 1.1 / 1.1.1）、有序 / 无序列表格式全书一致。

3. 图表、脚注、引用对齐

   图注、表注翻译完整且统一；章节交叉引用、参考文献、脚注编号不错乱。Markdown格式图片引用正确。

# 五、细节兜底：查漏补缺，消灭低级错误

1. 核对漏译、多译、错字

   重点看：小标题、括号内注释、页边备注、示例说明这些易被忽略的边角内容。

2. 人名、机构名、产品名统一

   国外厂商、开源项目、技术社区名称采用固定通用译名，不随意改写。

3. 上下文逻辑连贯

   已译完的前半部分，和后续章节的概念衔接、定义呼应，不自相矛盾。

# 六、校审实操建议

1. 第一轮：**纯技术核对**，只看技术是否准确，不管文笔；
2. 第二轮：**术语 + 格式统一**，对照术语表逐章修正；
3. 第三轮：**语句润色通读**，顺逻辑、改病句；
4. 第四轮：**细节终校**，查错别字、标点、格式瑕疵。
